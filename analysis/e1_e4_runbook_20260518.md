# E1-E4 HP-SPGG Experiment Runbook - 2026-05-18

This runbook is the concrete execution spec for E1-E4. Smoke outputs use `*_smoke` names. Overnight outputs use `results/e1_e4`, `analysis/E*.json`, `tables/E*.csv`, and `figs/E*.png`.

## E1: Posterior Concentration Rate per Backbone

Setup: HP-SPGG c19, four CloudGPT backbones, vanilla `hpsmg`, `K=20`, `seeds=5`, matched seeds, beta `0`.

Run: for each backbone calibration, rerun `llm_hpgg.run_experiment` with `--record-posterior --matched-seeds --algos hpsmg`. This writes `posterior_history[algo, seed, k, agent, type]` and `true_types[algo, seed, agent]` into the NPZ.

Analyze: `scripts/analyze_e1_posterior_concentration.py` extracts posterior mass on each agent's true type, computes mean trajectories across seeds and agents, and computes censored `K_conc^0.9 = min{k: all agents have true-type posterior >= 0.9}` per seed.

Plot: `figs/E1_posterior_concentration.png` has posterior trajectories, concentration-time bars, and a compact posterior-mass scatter. Tables are in `tables/E1_posterior_concentration.csv`; numeric summary is `analysis/E1_posterior_concentration_summary.json`.

## E2: Type Space Size Scaling

Setup: HP-SPGG scaling calibration, Llama backbone label/noise profile, `|Theta_i| in {2,3,4,5,6}`, `n=3`, `K=20`, `seeds=5`, algorithms `hpsmg` and `hpsmg_plus`, beta `0.25` for the plus bonus.

Run: `scripts/build_hp_spgg_scaling_calibration.py` creates coherent persona sets by progressively adding `selfish`, `cooperative`, `conditional_cooperator`, `altruistic`, `free_rider`, and `punisher`. Then `llm_hpgg.run_experiment` runs both algorithms with matched seeds and posterior traces.

Analyze: `scripts/analyze_e2_type_scaling.py` reads final cumulative regret at K for each type count and algorithm, computing mean and SEM.

Plot: `figs/E2_type_scaling.png` shows cumulative regret versus `|Theta_i|`. Tables are in `tables/E2_type_scaling.csv`; numeric summary is `analysis/E2_type_scaling_summary.json`.

## E3: n-Agent Scaling

Setup: HP-SPGG scaling calibration, Llama backbone label/noise profile, `n in {2,3,4,5}`, `|Theta_i|=4`, `K=20`, `seeds=5`, algorithms `hpsmg`, `joint_psrl`, `hpsmg_plus`.

Run: build one calibration per `n`, then run the three algorithms with matched seeds. Storage entries are recorded by `llm_hpgg.run_experiment`: factored H-PSMG uses `n|Theta_i|`, joint PSRL uses `|Theta_i|^n`.

Analyze: `scripts/analyze_e3_n_agent_scaling.py` reads final cumulative regret and storage from each NPZ.

Plot: `figs/E3_n_agent_scaling.png` has performance and log-scale storage panels. Tables are in `tables/E3_n_agent_scaling.csv`; numeric summary is `analysis/E3_n_agent_scaling_summary.json`.

## E4: Adversarial Prior Recovery

Setup: HP-SPGG c19 DeepSeek calibration, priors `{uniform, correct, adversarial}`, prior mass `0.7`, algorithms `hpsmg` and `map_greedy`, `K=20`, `seeds=5`, matched seeds.

Run: `llm_hpgg.run_experiment` uses `--prior-mode` and `--prior-mass`. Correct prior puts 0.7 mass on the true persona; adversarial prior puts 0.7 mass on the next wrong persona. Both factored and joint posterior states are initialized consistently.

Analyze: `scripts/analyze_e4_prior_recovery.py` computes cumulative regret curves and final means by prior and algorithm.

Plot: `figs/E4_prior_recovery.png` shows regret trajectories. Tables are in `tables/E4_prior_recovery.csv`; numeric summary is `analysis/E4_prior_recovery_summary.json`.
