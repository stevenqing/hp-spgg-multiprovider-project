# Data Provenance — HP-SPGG / H-PSMG$^+$ ARR Paper

Every numeric value, table cell, and figure data point in the ARR paper traces back to one JSON/CSV file under `analysis/` produced by exactly one experiment driver in the source tree. This document is the single source of truth for those mappings.

**Scope.** Only the data referenced by `arr_paper/main.tex` and `arr_paper/appendix.tex` is listed. Smoke tests, intermediate calibration shards, and exploratory variants live in `_archive/analysis_intermediate/` and are not part of the publication record.

**Backbones.** All "live" runs use the same four CloudGPT backbones unless noted:

| Short name | CloudGPT slug |
| --- | --- |
| GPT-5.4-nano | `gpt_5_4_nano_20260317` |
| DeepSeek-V3.2 | `DeepSeek_V3_2` |
| Kimi-K2.6 | `Kimi_K2_6` |
| Llama-4-Maverick | `Llama_4_Maverick_17B_128E_Instruct_FP8` |

**Concordia dependency caveat.** Concordia drivers import the upstream Concordia examples from `external/concordia/`. That folder was moved to `_archive/external/` by the May 2026 sweep; re-running any Concordia experiment requires restoring it:

```powershell
Move-Item _archive\external\concordia external\
```

The result JSONs themselves are unaffected and remain in `analysis/`.

**Concordia structural caveat.** The paper's headline panel uses the compact one-shot evaluators, not the full dialogue simulator. Official gullible/stubborn configs contain fixed supporting-player responses, but the compact Haggling runner does not execute those fields. See [concordia_structural_mapping.md](concordia_structural_mapping.md) for the TI/RL/PF and known-model audit.

**SOTOPIA structural caveat.** The SOTOPIA adapter is a broad out-of-model transfer, not a PF-only stress test. Cases are curated dyads, reward is a joint-goal LLM evaluator, actions are open text, and the four-class posterior uses keyword log-score increments rather than persona-conditioned next-token likelihoods. See [sotopia_structural_mapping.md](sotopia_structural_mapping.md) for the full assumption and oracle audit.

**Deployment-robustness experiment.** The main-text and appendix robustness tables are generated by `scripts/run_hp_spgg_deployment_robustness.py`. They use the analytic HP-SPGG reward surface with the mixed-backend noise profile (calibration seed 19), $n=3$, $|\Theta_i|=4$, $K=20$, $\beta=0.25$, likelihood width $\sigma=0.08$, and 500 paired seeds. The run makes zero LLM/API calls. It outputs `analysis/hp_spgg_deployment_robustness.{json,csv,md}`; the JSON retains all 8,500 per-seed records. Likelihood perturbations, fixed calibration drift, convex persona mixtures, and restricted candidate planning are controlled offline interventions. These results do not estimate SOTOPIA projection accuracy and are not four-backbone live reruns.

**AAAI-27 reviewer experiments.** All spec outputs live under `analysis/aaai27_review/` and are summarised by `PACT_AAAI27_REVIEWER_EXPERIMENTS.md`. E-R1 perturbs the actual analytic Gaussian outcome-likelihood interface because no retained token-logprob cache or logprob-enabled CloudGPT wrapper exists; it is not token-logprob evidence. E-R2 is an offline discrete-library diagnostic; prompt-baseline rows explicitly record non-applicability rather than fabricated comparisons. E-R3 uses public `cmu-lti/sotopia` data and raw per-cell JSON checkpoints; it runs only after correcting the adapter to read partner utterances from `Observation.last_turn`, and an episode is accepted only when no agent generation raised a caught provider-call exception. The counter does not identify malformed/invalid/missing action fields defaulted by the action parser or evaluator dimensions defaulted to zero, and the legacy cleanup schema did not retain discarded attempt history; final zero failure counts apply to accepted provider calls, not end-to-end provider/schema reliability. E-R4 uses restored public Concordia configs and analytic payoff functions. E-R0 reconstructs per-seed MaaSSim rows cache-first, records cache hits/live fills, and compares reconstructed aggregates with the retained scenario-suite aggregate CSV; it is not represented as recovery of deleted original seed rows. `scripts/validate_aaai27_reviewer_experiments.py` enforces the complete grids before release, and `scripts/package_aaai27_reviewer_experiments.py` creates the reviewer-only integrity bundle.

## Reproducibility checklist

To regenerate any single figure end-to-end:

1. Restore `external/concordia/` if the figure is a Concordia figure: `Move-Item _archive\external\concordia external\`.
2. Set `LLM_HPGG_BACKEND` and (if a live run) `LLM_HPGG_MODEL`.
3. Run the driver command(s) listed in the figure's row above, writing JSON to `analysis/`.
4. Run the producer script (`uv run python scripts\plot_fig_<name>.py`).
5. Copy the resulting PDF to `arr_paper/figs/` and `arr_paper_overleaf/figs/`.
6. Recompile the paper (see [../README.md](../README.md#recompiling-the-paper)).

If a row says *offline*, the producing script was archived because the figure is final; the PDF/PNG in `arr_paper/figs/` is canonical and does not need to be rebuilt.
